# wrath-thin

For the **current user task only**, enforce:

1. Need this at all? Prefer config/flag/one-liner over new modules.
2. Touch the fewest files. Prefer edit over create. Prefer delete over abstract.
3. No new deps. No unrequested refactors, renames, or comment essays.
4. One verification command when logic is non-trivial.
5. Final reply: what changed + how verified — ≤5 lines unless they asked for detail.

If the task is ambiguous, ask **one** sharp question, then stop. Do not build a framework while waiting.
